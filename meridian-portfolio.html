Problem & Solution

One Problem. Five Answers.

Each core feature of Meridian directly resolves a specific frustration uncovered in the discovery phase. Nothing was built for feature completeness alone.

Smart
Dashboard
One view for the entire portfolio. Metric cards, project table suggestions, and a mini timeline all above the fold. Reduces daily context-switching from 3 tools to 1.
AI
Prioritisation
Surfaces the 3 most critical actions every morning risk flags, workload imbalances, and sprint opportunities. PMs act on suggestions 73% of the time, validated in testing.
Core Problem
Enterprise PMs lose hours daily to cognitive overload
Existing tools surface too much, prioritise nothing, and require multiple apps to answer one question: "What do I need to do right now?"
5 Direct Solutions ↓
Gantt
Timeline
8-week rolling Gantt for all projects simultaneously. Hover any bar to see hours remaining no modal needed. Milestone diamonds, today marker, at-risk indicators.
Kanban
Board
Sprint-level task board with inline progress bars and priority colour-coding. Add tasks directly to the right status column zero-modal sprint management.
Mobile App Anywhere Access
Full Meridian experience on mobile task updates suggestions, timeline view, and notifications. Designed mobile-first, which also improved the desktop information hierarchy.
Self-Imposed Constraints
• 8-week timeline, solo designer · • WCAG AA (min 4.5:1 contrast) · • 8px grid system · • Enterprise info density (50+ tasks visible without scrolling) · • Desktop-first with mobile companion · • AI features non-blocking (suggestions, not commands) · • No proprietary project data

Testing Methodology
Moderated usability testing with 12 participants (ages 28–45, all senior PMs or engineering leads managing 10+ person teams) using a clickable Figma prototype. Each session was 45 minutes. Participants attempted 5 core tasks: (1) Create a task and assign it with priority, (2) Switch from Kanban to Gantt view and find an overdue item, (3) Review an AI priority suggestion and accept or dismiss it, (4) Find a blocked task and identify its dependency, (5) Check project status on mobile during a "commute" scenario. Task completion was binary (within 3 min). The "< 15s triage" was the median across all 12 participants on task 1.
Reflections & Trade-offs

What I'd Do Differently

Honest trade-offs and changes I'd make with more time, a team, or real enterprise data.

REFLECTION 01
AI Suggestions Need Trust Building
The AI panel suggests priority changes but 4 of 12 testers said they'd ignore AI recommendations without seeing how the model works. With more time: I'd add an "AI reasoning" expandable section showing why each suggestion was made, with a confidence score and the data points it used.
REFLECTION 02
Gantt View Doesn't Scale Past 50 Tasks
The Gantt timeline works beautifully for 20–30 tasks but becomes visually overwhelming at enterprise scale (200+ tasks). With more time: I'd design a collapsible grouping system (by team, milestone, or sprint) and a "critical path only" toggle that hides non-blocking tasks.
REFLECTION 03
Mobile Was a Late Addition
The mobile view works for status checks but can't handle task creation or Gantt manipulation. I designed it as a "read-mostly" companion, not a full PM tool. The lesson: Mobile-first would have forced better information hierarchy decisions that the desktop version now needs retroactively.
REFLECTION 04
Permissions & Roles Are Invisible
Enterprise PM tools need role-based access (PM, dev, stakeholder, viewer). Meridian currently shows the same UI to everyone. What I'd add: Role-specific dashboards stakeholders see status and burndown, developers see their assigned tasks + sprint backlog, PMs see the full orchestration view.
